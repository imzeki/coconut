Database Module Guide In Coconut

Function: createDatabase

Parameters: name

name:str -> The name of the database

createDatabase creates a python file inside the databases folder in Coconut and stores it in there. If there is not format given(e.g "Test" instead of "Test.py"), it will accept, however, if it ends with a different format like .js, it will raise InvalidTypeDatabaseError

Function: deleteDatabase

Parameters: name

name:str -> The name of the database to delete

deleteDatabase deletes the database name in the databases folder in Coconut. If no format is given, it will raise an error.

Function: readDatabase

Parameters: name

name:str -> The name of the database to read

readDatabase reads the database and returns the code written inside of the database.

Function: appendDatabase

Parameters: filename, code

filename:str -> The name of the database to append a line of code to.

code:str -> The code you want to append to the database, e.g. x = 10

appendDatabase appends a line of python code into the database, else, those files have no use!

Function: executeDatabase

Parameters: filename

filename:str -> The name of the file to execute.

executeDatabase executes the code inside the name of the database.

Class: NewCategory

Parameters: nameofcategory

nameofcategory:str -> The name of the category

Coconut's class NewCategory creates a new folder, known as Category in Coconut, inside there will be a __init__.py file for easier import. You can create subcategories which will be explain in createSubCategory

Function: createCategory

Parameters: None(self)

self:NewCategory -> to refer to itself

Creates the folder inside the databases folder. __init__.py file will be found inside and created from createInitFile()
